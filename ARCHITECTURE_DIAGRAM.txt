Development (Docker Compose)
├─ Single command: docker-compose up
├─ All services local
├─ Perfect for feature development
└─ sqlite backend for simplicity

Production (Kubernetes)
├─ Multi-node cluster
├─ Horizontal scaling (HPA)
├─ High availability
├─ Persistent volumes
├─ Monitoring + Alerting
└─ Managed PostgreSQL backend

╔════════════════════════════════════════════════════════════════════════════════╗
║ SCALABILITY                                                                     ║
╚════════════════════════════════════════════════════════════════════════════════╝

Horizontal Scaling (HPA)
─────────────────────
  Inference: 2-5 replicas (based on CPU 70%, Memory 80%)
  Gateway:   2-4 replicas (based on CPU 60%)
  
  ✓ Load balancing via service discovery
  ✓ Sticky sessions for auth state
  ✓ Request routing via labels

Vertical Scaling
───────────────
  CPU Request:   100m → 2000m (per service)
  Memory Request: 256Mi → 2Gi (per service)
  
  ✓ Defined resource limits
  ✓ Node affinity rules (optional)
  ✓ Pod priority classes (optional)

Cost Optimization
─────────────────
  ✓ Spot/Preemptible instances for training
  ✓ Scale to 0 during off-hours (via Cron HPA)
  ✓ Archive old MLflow runs to S3
  ✓ Managed services (RDS, Datadog) vs self-hosted
